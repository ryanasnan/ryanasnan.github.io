Mobile Apps¶

Intro¶

aplikasi pada gadget dan mobile saat ini tidak sepenuhnya menggunakan code dari program sepenuhnya

Ini dikarenakan adanya penggunaan API yang mana dapat mengirimkan data yang dibutuhkan.

API ini dibangun di program utama. Pengaksesannya tidak jauh berbeda, Pertama terdapat request di program utama selanjutnya program utama tersebut akan menghasilkan adanya data (response)

Nah di dalam aplikasi gadget/mobile inilah kita melakukan request dan mendapatkan response dari program utama. Sehingga tidak perlu membangun code dari aplikasi gadget/mobile tersebut, cukup dengan data yang didapatkan(response) akan diolah menjadi apa

Dari pengamatan penulis sepertinya program utama yang dijadikan sebagai mesin pengolah berasal dari pemrograman internet(php,js,asp.net) karena memang dengan menggunakan program internet kita dapat mengirim request dan menerima response jauh lebih mudah

dan ini juga kita tidak melulu hanya menggunakan adanya proses pengambilan(get) saja, karena kita juga dapat mengirim data (POST) bahkan menggunkana PUT/PATCH (edit data), dan masih banyak tipe lainnya

Integrasi dengan laravel¶

mobile apps dalam laravel kita mengenal adanya penggunaan token saat kita login token itu akan digunakan dalam mengautentikasi user dan ketika logout maka token itu akan berganti aku sebenarnya kurang tahu bagaimana pengaruh token dalam mekanisme autentikasi login/logout dalam laravel tetapi sepertinya dibutuhkan adanya token dalam mengakses suatu request, ini dipasang dalam middleware

well, lalu bagaimana pengintegrasian di dalam mobile apps untuk penggunaan laravel ini dibutuhkan adanya package “jwt.auth”

aku kurang tahu bagaimana mekanismenya yang jelas terdapat adanya penggunaan yang hampir sama

saat kita login kita akan menggeranate token (lewat package “jwt.auth”) jika data ada di database, maka kita akan mendapatkan token

selanjutnya gadget/mobile akan menyimpan token tersebut

kemudian setelahnya, ketika kita akan merequest dengan menyertakan adanya token tersebut dan di laravel terdapat adanya middleware yang menggunakan package “jwt.auth” selanjutnya package “jwt.auth” akan mengautentikasi token tersebut

token tersebut adalah berisikan user, sehingga kita tidak perlu menyimpan user_id,

namun perlu diketahui bahwa ketika kita mengirimkan suatu post dalam laravel maka dibutuhkan adanya token, token yang dimaksud disini adalah token untuk verify(verfycsrftoken) yaitu token saat akan post dalam laravel tentu ini akan menjadi masalah jika menggunakan nya, maksudnya hanya mengirim email dan password dalam form, karena masih dibutuhkan satu field bernama _token yang berisikan token agar dapat melakukan method post dalam laravel oleh karena itu sebaiknya kita berikan saja exception token dalam kasus ini

yaitu dengan menambahkan url route pada property except pada file “AppHttpMiddleware” seperti misalkan:

protected $except = [
        'api/authenticate',
    ];

tentu ini adalah untuk method post, penulis belum yakin apakah ini juga berlaku kepada method put/patch, karena ketika menggunakannya dalam laravel juga harus berada pada form dengan method POST namun dengan tambahan field _method bervalue “PUT” atau “PATCH” <input type=”hidden” name=”_method” value=”PUT”>
